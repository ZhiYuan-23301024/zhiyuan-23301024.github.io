    Integer缓存源码

    public static Integer valueOf(int i) {
    if (i >= IntegerCache.low && i <= IntegerCache.high)
        return IntegerCache.cache[i + (-IntegerCache.low)];
    return new Integer(i);
}
private static class IntegerCache {
    static final int low = -128;
    static final int high;
    static {
        // high value may be configured by property
        int h = 127;
    }
}

    # 接口和抽象类的区别

    一个类能实现多个接口，但只能继承一个父类

    接口的变量必须是public static final级别，而抽象类则不然

    接口比起抽象类更加的通用？适合做一些通用的功能实现，而抽象类更在乎的是代码的复用？

    就是说接口更像是一个给别人赋能的武器，而抽象类则是万千化身的源头？

    # 深拷贝和浅拷贝的区别

    浅拷贝：浅拷贝会在堆上创建一个新的对象（区别于引用拷贝的一点），不过，如果原对象内部的属性是引用类型的话，浅拷贝会直接复制内部对象的引用地址，也就是说拷贝对象和原对象共用同一个内部对象。

    深拷贝：深拷贝会完全复制整个对象，包括这个对象所包含的内部对象

    # hashCode()

    • equals 方法判断两个对象是相等的，那这两个对象的 hashCode 值也要相等。
    • 两个对象有相同的 hashCode 值，他们也不一定是相等的（哈希碰撞）。

    hashset使用的是hashcode判断两个元素是否重复

    和equal一样默认基于内存比较

    以上对于Object类成立

    abstract class AbstractStringBuilder implements Appendable, CharSequence {
    char[] value;
    public AbstractStringBuilder append(String str) {
        if (str == null)
            return appendNull();
        int len = str.length();
        ensureCapacityInternal(count + len);
        str.getChars(0, len, value, count);
        count += len;
        return this;
    }

    # 异常

    Exception是程序本身可以处理的异常，逻辑级别的错误，通过catch捕获，**也可以理解为局部错误？**处理妥当一般并不会影响到其余代码块

    Error是程序无法处理的错误，JVM级别的错误，jvm一般选择线程终止，一般影响范围是全局性的？

    Checked && Unchecked Exception 前者必须在编码的时候显式处理，即被catch或throws处理,后者则不需要

    try {
    System.out.println("Try to do something");
    throw new RuntimeException("RuntimeException");
} catch (Exception e) {
    System.out.println("Catch Exception -> " + e.getMessage());
} finally {
    System.out.println("Finally");
}
