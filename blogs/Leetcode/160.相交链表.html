    ZhiYuan 7/29/2025 双指针链表

    问题描述

    给定两个单链表的头节点 headA 和 headB，返回它们的第一个相交节点（节点地址相同，而非节点值相同）。若不相交，返回 null。

    关键概念：

    • 链表相交指从某一节点开始，后续节点地址完全一致（呈“Y”形结构）。
    • 需保留链表原始结构，且假设链表中无环。

    暴力解法

    核心思路：
    遍历链表A的每个节点，与链表B的所有节点进行地址比对，首个地址相同的节点即为交点。

    时间复杂度：

    • O(m×n)（m、n为链表长度），最坏需遍历所有节点组合。

    代码实现：

    public class Solution {
    public ListNode getIntersectionNode(ListNode headA, ListNode headB) {
        if (headA == null || headB == null) return null; // 边界检查

        ListNode pA = headA; 
        while (pA != null) {                 // 外层循环：遍历链表A
            ListNode pB = headB;             // 内层循环每次从头遍历链表B
            while (pB != null) {
                if (pA == pB) {              // ⚠️ 比较节点地址（非节点值！）
                    return pA;                // 找到交点
                }
                pB = pB.next;                // 移动链表B指针
            }
            pA = pA.next;                    // 移动链表A指针
        }
        return null;                          // 无交点
    }
}

    局限性：

    • 效率低：长链表（如 >1000 节点）时耗时剧增。
    • 易错点：误用 pA.val == pB.val 代替地址比较，可能导致值相同但地址不同的误判。

    双指针解法

    核心思路：

    1. 指针 p1 和 p2 分别遍历链表A和B；
    2. 当指针到达末尾时，重定向至另一链表的头部；
    3. 相遇条件：
      • 若相交，两指针走过相同路径长度（A独立段 + 公共段 + B独立段）后必在交点相遇；
      • 若不相交，两指针最终同时为 null 并终止循环。

    数学原理：

    • 设链表A独立段长为 a，链表B独立段长为 b，公共段长为 c。
    • 两指针路径长度均为 a + b + c，确保同步到达交点或 null。

    时间复杂度：

    • O(m + n)，仅需遍历两链表各一次。

    代码实现：

    public class Solution {
    public ListNode getIntersectionNode(ListNode headA, ListNode headB) {
        if (headA == null || headB == null) return null; // 边界检查
        ListNode p1 = headA, p2 = headB;
        
        while (p1 != p2) {
            // 指针到末尾时重定向至另一链表头部
            p1 = (p1 == null) ? headB : p1.next; 
            p2 = (p2 == null) ? headA : p2.next; 
        }
        return p1; // 返回交点或null
    }
}
