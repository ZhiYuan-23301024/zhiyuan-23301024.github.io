    55.跳跃游戏

    ZhiYuan 8/2/2025 贪心动态规划

    题目

    给你一个非负整数数组 nums ，你最初位于数组的 第一个下标 。数组中的每个元素代表你在该位置可以跳跃的最大长度。

    判断你是否能够到达最后一个下标，如果可以，返回 true ；否则，返回 false 。

    原想法（递归）

    先上源码

    class Solution {
    public boolean canJump(int[] nums) {
        return canJumpFrom(0, nums, new HashSet<>()); 
    }
    
    private boolean canJumpFrom(int pos, int[] nums, Set<Integer> visited) {
        if (pos >= nums.length - 1) return true;
        if (visited.contains(pos)) return false;
        visited.add(pos);
        
        int maxJump = Math.min(pos + nums[pos], nums.length - 1);
        for (int next = maxJump; next > pos; next--) {
            if (canJumpFrom(next, nums, visited)) return true;
        }
        return false;
    }
}

    原先的想法是将问题抽象为首尾是否可达，那么我可以先看哪些下标（a,b,c...）可以直达nums尾元素，然后判断首元素是否可以抵达这戏下标，这样递归下去，将问题变为首元素是否可以抵达a,b,c...。然而这样会导致递归深度过大，导致栈溢出报错。

    贪心+动态规划

    class Solution {
public boolean canJump(int[] nums) {
    int maxReach = 0;
    for (int i = 0; i < nums.length; i++) {
        if (i > maxReach) return false;  // 当前位置不可达
        maxReach = Math.max(maxReach, i + nums[i]);
        if (maxReach >= nums.length - 1) return true;  // 提前终止
    }
    return true;
}
}

    这种解法将问题抽象为，若我只能从首节点到下标i-1这段路程中跳动，那么我能最多跳到哪里呢？

    如果我最多能跳到的位置小于i，那么很明显首节点和下标i之间是不可达的，那么自然下标i之后的所有位置都不可能抵达

    因此我们的动态规划就是动态地判断下标i是否可以抵达，该过程中借用贪心的性质进行放缩推理

    因此可以抽象出一条递推公式：maxReach(i) = max{ maxReach(i-1), i + nums[i] }

    该方法的时间复杂度为O(n)，空间复杂度为O(1)
