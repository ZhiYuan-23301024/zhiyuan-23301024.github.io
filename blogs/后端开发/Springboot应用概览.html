    什么是maven

    一、Maven 的核心定义与核心作用

    Maven 是 Apache 基金会维护的 Java 项目管理工具，主要用于 自动化构建、依赖管理 和 项目标准化。其核心特点包括：

    1. 项目对象模型（POM）
      通过 XML 文件（pom.xml）定义项目的依赖、插件、构建流程等信息，实现配置与代码的分离。
    2. 依赖管理
      自动从远程仓库下载所需的第三方库（如 Spring、JUnit），并递归处理依赖关系，避免手动管理 JAR 包的繁琐。例如，若项目依赖 A.jar，而 A.jar 又依赖 B.jar，Maven 会一并下载并配置。
    3. 标准化构建流程
      提供预定义的构建生命周期（如 compile、test、package），通过命令（如 mvn install）自动执行编译、测试、打包等操作。

    二、本地仓库的配置与核心作用

    本地仓库（Local Repository） 是 Maven 在开发者计算机上的 依赖缓存目录，其核心意义在于：

    1. 依赖存储机制

      • 默认路径：用户目录/.m2/repository（例如 Windows 的 C:\Users\用户名\.m2\repository）。
      • 功能：存储从远程仓库下载的 JAR 包，避免重复下载。例如，首次构建项目时会下载依赖到本地，后续构建直接使用缓存。

    2. 配置方法
      通过修改 Maven 的 settings.xml 文件（位于 Maven安装目录/conf/ 或用户目录的 .m2/ 下）中的 <localRepository> 标签，可自定义路径：

      <settings>
  <localRepository>D:\my-maven-repo</localRepository>
</settings>

      此配置适用于需要将仓库存储在非系统盘或团队统一路径的场景。

    3. 依赖查找优先级
      Maven 在构建时按以下顺序查找依赖：

      • 本地仓库 → 中央仓库（默认远程仓库） → 自定义远程仓库（如公司私服）。若本地已有依赖，直接使用；否则从远程下载并缓存到本地。

    三、本地仓库与远程仓库的关系

    对比维度 本地仓库 远程仓库
    存储位置 开发者本地计算机 远程服务器（如中央仓库、公司私服）
    访问速度 高速（无需网络） 依赖网络带宽（可能需配置镜像加速）
    更新机制 手动删除或通过 mvn -U 强制更新 自动下载新版本（SNAPSHOT 版本会实时更新）
    典型用途 缓存依赖，减少重复下载 提供公共或私有依赖的集中存储

    四、实际应用场景与问题解决

    1. 依赖冲突
      当多个依赖引用了不同版本的同一库时，可通过 <dependencyManagement> 统一版本号。

    2. 离线构建
      使用 mvn -o 命令可在无网络时基于本地仓库构建项目。

    3. 镜像加速
      在 settings.xml 中配置阿里云镜像，替换默认中央仓库以加速下载：

      <mirror>
  <id>aliyun</id>
  <mirrorOf>central</mirrorOf>
  <url>https://maven.aliyun.com/repository/public</url>
</mirror>

    总结

    Maven 通过 POM 模型 和 仓库体系 简化了 Java 项目管理，而本地仓库作为其核心组件，显著提升了构建效率。合理配置本地仓库路径（如避免系统盘空间不足）和镜像仓库，可进一步优化开发体验。

      HTTP协议（超文本传输协议）

      • 基于TCP协议
      • 基于请求-响应模型
      • 是无状态的协议，多次请求之间不能共享数据

      请求数据格式

      • 请求行：描述请求的方式、路径和协议版本

        GET /brand/findALL?name=OPPO&status=1 HTTP/1.1
        1
        POST /brand HTTP/1.1
        1

      • 请求头：若干个键值对

        • Host:请求的主机名
        • User-Agent:浏览器版本
        • Accept:标识浏览器能接受的资源类型
        • Accept-Language:浏览器偏好的语言
        • Accept-Encoding:浏览器可以接受的压缩类型
        • Content-Type:请求主题的数据类型
        • Content-Length:请求主体的大小（字节）

      • 请求体：POST请求，存放请求参数

        {
    "status":1,
    "brandName":"黑马",
    "companyName":"黑马程序员"
}

      请求数据获取

      服务器自动获取请求数据并封装到一个HTTPServletRequest对象里

      package com.example.demo;

import jakarta.servlet.http.HttpServletRequest;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class RequestController {
    @RequestMapping("/request")
    public String request(HttpServletRequest request) {
        //获取请求方式
        System.out.println(request.getMethod());
        //获取请求的url地址
        System.out.println(request.getRequestURL().toString());
        //获取请求协议
        System.out.println(request.getProtocol());
        //获取请求参数 -name
        System.out.println(request.getParameter("name"));
        //获取请求头 -Accept
        System.out.println(request.getHeader("Accept"));
        return "OK";
    }
}

      响应数据格式

      • 响应行：协议、状态码和描述

        HTTP/1.1 200 OK
        1

        状态码：

        • 1XX：响应中临时状态码标识请求已经被接受，提示下一步操作
        • 2XX：标识请求被成功接受并且被处理完毕
        • 3XX：重定向，要求浏览器再发起一次请求
        • 4XX：客户端错误，访问资源不存在，禁止访问等
        • 5XX：服务端错误，比如服务端做了非法操作

      • 响应头：键值对

      • 响应体：json数据

      SpringBootWeb案例

      package com.example.demo.controller;

import cn.hutool.core.io.IoUtil;
import com.example.demo.pojo.User;
import org.apache.tomcat.util.http.fileupload.IOUtils;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

import java.io.File;
import java.io.FileInputStream;
import java.io.InputStream;
import java.nio.charset.StandardCharsets;
import java.time.LocalDateTime;
import java.time.format.DateTimeFormatter;
import java.util.ArrayList;
import java.util.List;

@RestController
public class UserController {
    @RequestMapping("/list")
    public List<User> list(){
        //加载并读取数据文件
        InputStream in =this.getClass().getClassLoader().getResourceAsStream("user.txt");
        ArrayList<String> lines=IoUtil.readLines(in, StandardCharsets.UTF_8,new ArrayList<>());
        //解析数据
        List<User> userList=lines.stream().map(line->{
            String[] parts=line.split(",");
            Integer id=Integer.parseInt(parts[0]);
            String username=parts[1];
            String password=parts[2];
            String name=parts[3];
            Integer age=Integer.parseInt(parts[4]);
            LocalDateTime updateTime=LocalDateTime.parse(parts[5], DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss"));
            return new User(id,username,password,name,age,updateTime);
        }).toList();

        return  userList;
    }
}

    分层解耦

    web开发三层架构

    • Dao:数据访问，统一的接口和规范，面向接口编程
    • Service:逻辑处理
    • Controller:接收请求，响应数据

    分层解耦概念

    • 耦合：衡量软件中各个层/各个模块的依赖关联程度
    • 内聚：软件中各个功能模块内部的功能联系

    软件设计原则：高内聚低耦合

    IOC和DI入门

    用容器储存服务类、DAO类等类产生的 对象，调用的时候从容器中取

    • 控制反转：简称IOC，对象的控制权由程序自身转移到外部容器
    • 依赖注入：简称DI，容器为应用程序提供运行时依赖的资源
    • Bean对象：IOC容器中创建、管理的对象

    入门程序

    • 将Dao和Service的实现类交给IOC容器处理

      加入@Component标签

    • 为Controller和Service注入运行时依赖的对象

      加入@Autowired标签

    IOC详解

    • 要把某个对象交给IOC容器处理，需要在对应的类上加入如下注解之一

      注解 说明 位置
      @Component 声明bean的基础注解 不属于以下三类时使用
      @Controller @Component的衍生注解 标注在控制层上
      @Service @Component的衍生注解 标注在业务层上
      @Repository @Component的衍生注解 标注在数据访问层上（由于与mybatis结合，用的少）

      IOC中每一个bean都有名字，默认类名（首字母小写）

      自定义为@Component("name")

    • 如果bean的注解想要生效，还需要被组件扫描注解@ComponentScan扫描到，该注解没有显式的配置，但是实际上包含在启动类声明注解@SpringBootApplication中，默认扫描范围是启动类所在包和其子包

    package org.example.staffmanage.pojo;

import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;

@Data
@AllArgsConstructor
@NoArgsConstructor
public class Result {
    private int code;
    private String msg;
    private Object data;

    public static Result success() {
        Result result = new Result();
        result.code=1;
        result.msg="success";
        return result;
    }

    public static Result success(Object data) {
        Result result = new Result();
        result.code=1;
        result.msg="success";
        result.data = data;
        return result;
    }

    public static Result error(String msg) {
        Result result = new Result();
        result.code=0;
        result.msg=msg;
        return result;
    }
}

    API规范

    knife4j插件

    在配置类配置好拦截器和静态资源映射

    /**
 * 配置类，注册web层相关组件
 */
@Configuration
@Slf4j
public class WebMvcConfiguration extends WebMvcConfigurationSupport {

    @Autowired
    private JwtTokenAdminInterceptor jwtTokenAdminInterceptor;

    /**
     * 注册自定义拦截器
     *
     * @param registry
     */
    protected void addInterceptors(InterceptorRegistry registry) {
        log.info("开始注册自定义拦截器...");
        registry.addInterceptor(jwtTokenAdminInterceptor)
                .addPathPatterns("/admin/**")
                .excludePathPatterns("/admin/employee/login");
    }

    /**
     * 通过knife4j生成接口文档
     * @return
     */
    @Bean
    public Docket docket() {
        ApiInfo apiInfo = new ApiInfoBuilder()
                .title("苍穹外卖项目接口文档")
                .version("2.0")
                .description("苍穹外卖项目接口文档")
                .build();
        Docket docket = new Docket(DocumentationType.SWAGGER_2)
                .apiInfo(apiInfo)
                .select()
                .apis(RequestHandlerSelectors.basePackage("com.sky.controller"))
                .paths(PathSelectors.any())
                .build();
        return docket;
    }

    /**
     * 设置静态资源映射
     * @param registry
     */
    protected void addResourceHandlers(ResourceHandlerRegistry registry) {
        log.info("启动静态资源映射");
        registry.addResourceHandler("/doc.html").addResourceLocations("classpath:/META-INF/resources/");
        registry.addResourceHandler("/webjars/**").addResourceLocations("classpath:/META-INF/resources/webjars/");
    }
}
