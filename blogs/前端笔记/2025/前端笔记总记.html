    使用步骤

    • 创建路由组件的组件

    • 配置路由映射：组件和路径映射关系的routes数组

    • 通过createRouter创建路由对象，并且传入routes和history模式

      //router/index.js
import { createRouter,createWebHashHistory } from "vue-router";
import Homeview from "../view/Homeview.vue";
import Aboutview from "../view/Aboutview.vue";

//配置信息中需要页面的相关配置

const routes=[
    {
        path:"/",
        component:Homeview
    },
    {
        path:"/about",
        component:Aboutview
    }
]


const router=createRouter({
    routes,
    history:createWebHashHistory()
})

export default router

    组件通信props

    在Vue.js中，Props（Properties）是组件间通信的核心机制，用于实现父组件向子组件传递数据。以下从技术原理、应用场景和最佳实践三个维度详细解析：

    一、核心概念与原理

    1. 定义与作用
      Props是子组件接收父组件数据的自定义属性，通过声明式数据流实现组件解耦。父组件通过标签属性传递数据，子组件通过props选项接收并渲染。例如：

      <!-- 父组件 -->
<ChildComponent :message="parentData" />

<!-- 子组件 -->
<script>
export default {
  props: ['message']
}
</script>

    2. 单向数据流
      数据只能从父组件流向子组件，子组件不可直接修改props数据。若需反向通信，需通过$emit触发事件，父组件监听并更新数据。

    二、关键特性与使用方式

    1. 数据传递机制

    • 静态传递：直接传递字面量值（如title="Vue教程"）

    • 动态绑定：使用v-bind（或简写:）绑定父组件数据：

      <ChildComponent :posts="dynamicPosts" />
      1

      父组件数据变化时，子组件自动更新。

    2. 类型验证与约束

    通过对象语法定义props时，可指定类型、默认值和验证规则：

    props: {
  title: {
    type: String,
    required: true,
    default: '默认标题',
    validator: (value) => value.length <= 50
  }
}
    1

    • 支持类型：String、Number、Boolean、Array、Object等
    • 验证失败：控制台输出警告，避免错误数据传递

    3. 默认值与必填项

    • default：未传递数据时的回退值
    • required: true：强制要求父组件必须传递该prop

    三、高级应用与最佳实践

    1. 对象/数组的引用传递

    当传递对象或数组时，子组件接收的是引用。若直接修改其属性，会违反单向数据流原则，应通过深拷贝或事件通信更新。

    2. 动态双向绑定

    通过.sync修饰符实现类似双向绑定的效果：

    <!-- 父组件 -->
<ChildComponent :title.sync="parentTitle" />

<!-- 子组件 -->
this.$emit('update:title', newTitle)

    3. 性能优化

    • 避免深层嵌套：复杂对象props可能导致不必要的渲染
    • 纯函数式组件：对无状态组件使用functional: true提升性能

    四、典型应用场景

    场景 实现方案 优势
    列表项渲染 父组件传递数组，子组件循环渲染 复用性强，逻辑隔离
    表单控件封装 传递初始值，通过事件回传修改 统一校验逻辑，降低耦合度
    配置化组件 通过props接收UI配置参数（如主题色） 灵活定制，扩展性高

    Vue渲染markdown

    Vue.js 完全支持在 Web 端渲染 Markdown，且可通过多种主流方案实现。以下是综合多种实现方式的总结及核心方法：

    一、使用第三方 Markdown 解析库（主流方案）

    1. marked 或 markdown-it 库

    • 实现方式：通过安装 marked 或 markdown-it 库解析 Markdown 文本，结合 Vue 的 v-html 指令渲染 HTML 内容。

    • 核心代码示例：

      <template>
  <div v-html="renderedMarkdown"></div>
</template>
<script>
import { marked } from 'marked';
export default {
  data() {
    return { markdown: '# Hello World' };
  },
  computed: {
    renderedMarkdown() {
      return marked(this.markdown);
    }
  }
};
</script>

      • 全局引入：可通过 Vue Mixins 全局注册，避免重复导入。

    2. markdown-it-vue 库

    • 功能特性：专为 Vue 设计的 Markdown 组件，支持 TOC（目录生成）、Mermaid 图表、流程图等高级功能。

    • 使用示例：

</template>
<script>
import MarkdownItVue from 'markdown-it-vue';
export default {
  components: { MarkdownItVue },
  data() { return { markdownText: '## Content' }; }
};

    三、插件化方案（增强功能）

    1. Vue-Markdown

    • 特点：轻量级插件，适合快速渲染文档。
    • 安装：npm install vue-markdown，支持按需引入。

    2. Toast-ui.vue-editor

    • 功能：集成编辑器与预览模式，支持数学公式和图表。
    • 适用场景：需要富文本编辑与实时预览的复杂项目。

    注意事项

    1. XSS 安全：使用 v-html 渲染时需确保内容可信，避免注入攻击。
    2. 样式优化：引入 CSS 文件（如 github-markdown-css）优化渲染效果。
    3. 性能对比：markdown-it 和 marked 性能较高，适合大型文档；markdown-it-vue 适合需要深度定制的场景。

    总结

    Vue.js 在 Web 端渲染 Markdown 的方案多样，开发者可根据需求选择：

    • 基础场景：marked 或 markdown-it + v-html。
    • 复杂需求：markdown-it-vue 或 vue-markdown-loader。
    • 编辑器集成：Toast-ui.vue-editor 或 Vue-Simplemde。

    具体实现可参考各库的官方文档和示例代码。
