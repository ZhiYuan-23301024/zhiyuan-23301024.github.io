    # GNN相关

    点的特征变化由相邻点映射而来（参考全链接？）

    这个过程输入图结构不变化

    GNN只负责提取有用的特征，具体的分类和回归啥的还是要将数据降维到我们的传统CNN这块？

    那么为什么不直接用传统的呢？传统的也可以提取特征

    GNN相比之下的好处在于可以处理规模会变的数据

    社交网络啊啥的这种东西，每次处理的数据规模不一样

    但是传统的cnn它是要resize的

    嵌入向量

    将本来的图元素依照其特征映射到一个低维连续向量上，同时希望这个向量通过某种距离函数能反映原图中的拓扑结构，比如关系远近？

    提示和输出

    • 提示：图数据+任务目标
    • 输出：根据任务不同而不同

    预训练和微调

    • 预训练：海量无标签数据，通用学习
    • 微调：较少特定数据，针对学习

    无标签不意味着**“瞎几把乱学”**，而是标签是模型自己创造的而非人为标定

    # 如何缝合

    • 尽可能找到有价值的代码（可以直接拿来复用或改动代价小的），比如两个项目里基于nn.module实现的子类，这些东西的ipo过程相对固定，没有什么所谓“自己的想法”，比较好缝合，而针对某个数据集做的一些util或者说process属于个人特色比较鲜明的，不要去尝试缝合，会变得不幸
    • 在搜集到了有价值的代码后，确定好缝合方向，再根据原论文的设计指导，利用有价值的代码碎块，搭建自己的GNN训练工作流。

    # HG高价值代码

    # conv

    • conv.myGATconv:自定义的图注意力卷积层

    # GNN

    • GNN.myGAT:基于myGATconv搭建的异构图多层GAT模型
    • GNN.GCN:基于标准 GCN 层专门为异构图学习设计的神经网络模块。负责学习节点的表示，但分类头需要在模型外部定义
    • GNN.GIN:图同构网络模型，对照组
    • GNN.GAT:标准图注意力卷积层，对照组

    # hprompt

    • node_prompt_layer_feature_cat:节点提示层，将可学习的提示向量拼接到每个节点的现有嵌入上
    • node_prompt_layer_feature_cat_edge:应用在边上

    上面那俩是同构图

    • hnode_prompt_layer_feature_cat_edge
    • hnode_prompt_layer_feature_cat

    这俩是支持异构图

    • hprompt_gcn:异构图卷积层
    • hnode_prompt_layer_feature_sum:异构图特征聚合层

    # GP高价值代码

    • base_model:基础模型，GraphPro的子类
    • GraphPro:处理时序图上推荐任务的图神经网络（GNN）模型。继承自 BaseModel 基类
    • finetune:微调GraphPro脚本
    • GraphProPlugin:GraphPro作为插件的实现
    • dynamicGNN：动态 GNN
    • graphprompt :提示 GNN
    • LightGCN ：GraphPro的阉割版，作对照组用
