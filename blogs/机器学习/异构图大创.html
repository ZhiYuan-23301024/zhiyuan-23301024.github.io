    数据处理流程中的灵活性​
    在数据处理的中间步骤，特别是数据预处理、清洗和格式转换阶段，TXT 文件因其易于用常见命令行工具（如 grep, sed, awk）和脚本（Python 等）处理而备受青睐 。许多历史悠久的经典图数据集​（如某些社交网络、引文网络）最初就以 TXT 格式发布，为了保持一致性，后续研究常会沿用。 若从高效的角度来考虑应该使用.pt和.npy文件

    本来想做完整的3个G的图数据预处理来着但是内存干爆了 就采用了3%数据量 整个过程的目标是构建能够直接被pytorch反序列化的.pt用于存储边权和节点特征，而.npy文件用于存储节点id、边id到原始id的映射规则，也就是说，如果只需要训练并统计模型性能，那么理论上只需要.pt就行了 这种结构的好处是可以直接调用，在项目里无需进行数据预处理了，换句话说这本身就是数据预处理

# 2. ==================== GNN模型定义 ====================
class UserItemGNN(torch.nn.Module):
    """
    用户-物品交互图神经网络模型
    """
    def __init__(self, in_channels, hidden_channels, out_channels):
        super(UserItemGNN, self).__init__()
        self.conv1 = GCNConv(in_channels, hidden_channels)
        self.conv2 = GCNConv(hidden_channels, out_channels)
        self.dropout = nn.Dropout(0.5)
        
    def forward(self, x, edge_index, edge_weight=None):
        # 第一层图卷积
        x = self.conv1(x, edge_index, edge_weight)
        x = F.relu(x)
        x = self.dropout(x)
        
        # 第二层图卷积
        x = self.conv2(x, edge_index, edge_weight)
        
        return x

# 3. ==================== 训练函数 ====================
def train_model(data, model, optimizer, criterion, epochs=100):
    """
    训练GNN模型
    """
    model.train()
    losses = []
    
    for epoch in range(epochs):
        optimizer.zero_grad()
        
        # 前向传播 [使用数据集中的节点特征和边索引]
        out = model(data.x, data.edge_index, data.edge_attr)
        
        # 这里需要根据你的任务定义标签和损失函数
        # 假设是节点分类任务，你需要有自己的标签数据
        # loss = criterion(out[data.train_mask], data.y[data.train_mask])
        
        # 由于你没有提供标签，这里使用一个伪损失函数示例
        # 实际使用时需要替换为你的真实标签
        pseudo_loss = torch.mean(out) * 0.001  # 这行需要替换
        
        pseudo_loss.backward()
        optimizer.step()
        
        losses.append(pseudo_loss.item())
        
        if epoch % 10 == 0:
            print(f'Epoch {epoch:03d}, Loss: {pseudo_loss.item():.4f}')
    
    return losses


# 5. ==================== 主函数 ====================
def main():
    # 设置设备
    device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
    print(f"使用设备: {device}")
    
    # 加载数据 [主要数据集调用]
    data, user_mapping, item_mapping = load_processed_data('processed_data')

    print("开始训练模型...")
    # 训练模型 [使用数据集进行训练]
    losses = train_model(data, model, optimizer, criterion, epochs=50)
    
    print("开始评估模型...")
    # 评估模型 [使用数据集进行评估]
    outputs = evaluate_model(data, model)

    plt.ylabel('Loss')
    plt.title('Training Loss Over Time')
    plt.legend()
    plt.savefig('training_loss.png')

    print("模型训练和评估完成!")
    print("可以进一步进行:")
    print("1. 节点表示学习（用于推荐）")
    print("2. 链接预测（预测潜在交互）")
    print("3. 用户/物品分类")

if __name__ == "__main__":

    整个过程可以概括为：将原始数据读入内存 → 在内存中构建图数据对象 → 将图数据对象序列化后保存到硬盘。

    阶段 关键任务 内存使用特点
    1. 原始数据读取 将磁盘上的原始数据（如CSV）加载到内存中的数据结构（如Pandas DataFrame） 内存占用高峰之一，数据完全载入内存
    2. 内存中数据处理 数据清洗、转换、构建图结构（节点、边、特征）、映射关系等 内存占用高峰之二，原始数据与图结构同时存在
    3. 序列化与保存 将内存中的图数据对象转换为字节流，写入硬盘上的特定格式文件（如.pt） 内存占用开始下降，数据从内存转移到磁盘

    (80 封私信 / 80 条消息) 图分割Graph Partitioning技术总结 - 知乎 (opens new window) 将一个大图均匀的分成一系列的子图去适应分布式应用，每个子图存储在一台机器上，子图之间可以并行化执行，如果当前子图需要其他子图的信息就需要通讯开销，而图分割的质量影响着每台机器存储代价和机器之间通讯代价。

    图分割的两个目标是负载均衡减少存储代价）和最小化切边或点(减少通讯代价），同时优化这两个目标是平衡图分割问题。

def save_experiment_record(experiment_type, params, results):
    """
    保存实验记录到CSV文件
    :param experiment_type: 实验类型 (e.g., "single_graph", "distribution", "multi_graph")
    :param params: 实验参数字典
    :param results: 实验结果字典
    """
    # 创建唯一时间戳
    timestamp = datetime.datetime.now().strftime("%Y%m%d_%H%M%S")
    
    # 创建记录字典
    record = {
        "experiment_id": f"{experiment_type}_{timestamp}",
        "timestamp": timestamp,
        "experiment_type": experiment_type,
    }
    
    # 添加参数
    record.update({f"param_{k}": v for k, v in params.items()})
    
    # 添加结果
    record.update({f"result_{k}": v for k, v in results.items()})
    
    # 创建DataFrame
    df = pd.DataFrame([record])
    
    # 保存文件
    filename = f"experiment_records/{experiment_type}_records.csv"
    
    # 如果文件存在则追加，否则创建新文件
    if os.path.exists(filename):
        existing_df = pd.read_csv(filename)
        df = pd.concat([existing_df, df], ignore_index=True)
    
    df.to_csv(filename, index=False)
    print(f"✅ 实验记录已保存至: {filename}")

def save_results_to_csv(results):
    timestamp = datetime.datetime.now().strftime("%Y%m%d_%H%M%S")
    filename = f"result/distribution_analysis_{timestamp}.csv"
    data_to_save = {
        "loss": [results["mean_loss"]],
        "variance": [results["variance"]],
    }
    df = pd.DataFrame(data_to_save)
    df.to_csv(filename, index=False)
    print(f"Results saved to {filename}")

    • gnn_models:
      • GCN：核心特点在于通过奇异值分解（SVD）动态调整模型权重矩阵的秩（rank），用于控制模型复杂度
        • 输入输出隐藏层维度控制一致，无偏置项
        • 全局加和池化
        • 存储**当前模型每一层权重矩阵的实际秩**
      • GAT：与经典GAT区别同上，为每一层设定了一个目标秩 tar_rank，计算公式为 input_dim - n_rank。每一层的权重矩阵将输入特征投影到一个**维度降低了n_rank的子空间强制性的维度压缩或复杂度控制意图。图。==

    总而言之，无论是GCN还是GAT，都是再模型的构建过程利用奇异值分解算法有意识地控制各个层权重矩阵的秩为特值，这是和经典GCN和GAT的最本质区别

    • graph_generation

      • generate_random_graph：生成随机图数据（节点特征维度、链接概率，节点数量上下限）
      • graph_generate
      • 'S': 生成完全随机合成的图。
      • 'R': 从真实数据集中加载图。
      • 'B': ​混合模式，随机选择上述两种方式之一。

    • graph_matching：

      • graph_matching： 计算图转换和图提示结果之差，对差值计算L2范数，取均值

    作为核心的损失函数，直接衡量并引导优化“经过变换的图表示”与“添加了提示的图表示”之间的差异
